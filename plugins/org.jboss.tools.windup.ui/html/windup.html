Getting Started with the Migration Toolkit for Runtimes (MTR) Eclipse Plugin

Identify and Resolve Migration Issues with MTR

Run MTR against your project to identify and fix migration issues.

  1. Import the project to analyze.

  2. Create a MTR run configuration. From the Issue Explorer, press the MTR button ().

    At a minimum, select the project to analyze. Set additional options as needed.

  3. Click Run to execute MTR.

  4. Review MTR issues listed in the Issue Explorer.

  5. Resolve MTR issues by manually updating code or by using quick fixes when available.

  6. Run MTR again as necessary. Use the drop down next to the Run button () to run an existing configuration.

Create a MTR Run Configuration

A MTR run configuration can be created using the MTR button (). You can create multiple run configurations, each with a unique name.

Input

Select a migration path. This determines which MTR rulesets are used. The migration path defaults to Anything to EAP 7, but can be changed to any supported migration path.

Select one or more projects to analyze. Hold the Ctrl key to select multiple projects in the list.

Select one or more packages to scan. It is recommended to select only those packages that you need to analyze to reduce the overall MTR execution time. If no packages are selected, all packages in the project will be scanned. Hold the Ctrl key to select multiple packages in the list.

Options

Generate a MTR report. Check the Generate Report checkbox if you want to generate the MTR HTML report. The report will be shown in the MTR Report tab and can be found in the Issue Explorer when you group by File.

Set additional MTR options. Any option that is a boolean flag, such as enableTattletale, should enter true as the value. See the MTR User Guide for a description of each MTR argument.

Rules

Add a custom rules directory if you have any custom MTR rules.

Execute MTR

Once a run configuration has been created, you can execute MTR using that configuration in one of the following ways:

  • Select the run configuration from the Run Configurations dialog and click Run.
  • Select a recent run configuration from the drop down next to the Run button ().

If the Windup is not currently running, it will start once a run configuration is executed. If you do not plan on running MTR again in the near future, it is recommended to stop the Windup to conserve memory on your machine.

Review MTR Issues

Use the Issue Explorer to review migration issues identified by MTR. Different icons indicate the issue's severity and state.

Change how issues are grouped by adjusting the Group By selections: Severity, Migration Rule, File.

Double-click the MTR issue in the Issue Explorer to open the associated line of code in an editor. Right-click and select Issue Details to view information about the MTR issue, including its severity and how to address it.

Resolve MTR Issues

You can resolve MTR issues by updating the code manually or by applying a quick fix when available.

Manual Update

Review the MTR issue details and additional resources and update the source code as necessary. When you update a line of code marked as a MTR issue, the MTR issue will be marked with the stale icon () until the next time that MTR is run on the project.

You can also manually mark a MTR issue as fixed, which will mark the issue with the resolved icon () until the next time that MTR is run on the project. To mark an issue as fixed, right-click the MTR issue in the Issue Explorer and select Mark as Fixed.

Quick Fix

Some MTR issues provide a quick fix, which assists in making the necessary edits to address the issue.

Preview a Quick Fix

Right-click the issue and select Preview Quick Fix. This will bring up a window that allows you to preview the change. From here, you can apply the fix or close the window.

Apply a Quick Fix

Right-click the issue and select Apply Quick Fix. This will update the source code as required and will mark the MTR issue as resolved.

MTR Issue Legend

In the Issue Explorer and file editors, MTR issues use an icon to indicate their severity level and status. The following table describes the meaning of the various icons.

Icon Description
The issue is mandatory to fix for a successful migration.
The issue is optional to fix for migration.
The issue may potentially be an issue during migration.
The issue has been resolved.
The issue is stale because the code marked as an issue has been modified since the last time that MTR was run on the project.
A quick fix is available for this issue, which is mandatory to fix for a successful migration.
A quick fix is available for this issue, which is optional to fix for migration.
A quick fix is available for this issue, which may potentially be an issue during migration.
